INSTALLAZIONE SUL VPS
===============================================================================

1. TRASFERISCI L'ARCHIVIO SUL VPS
   ---------------------------------
   Dal tuo computer:
   
   scp proxy-dashboard-complete.tar.gz root@IP_VPS:~/

2. ESTRAI L'ARCHIVIO
   ------------------
   Sul VPS (via SSH):
   
   ssh root@IP_VPS
   
   # Fai backup del vecchio (se esiste)
   mv ~/proxy-dashboard ~/proxy-dashboard-backup-$(date +%Y%m%d)
   
   # Crea directory e estrai
   mkdir -p ~/proxy-dashboard
   tar -xzf proxy-dashboard-complete.tar.gz -C ~/proxy-dashboard
   cd ~/proxy-dashboard

3. INSTALLA DIPENDENZE
   --------------------
   npm install

4. AVVIA LA DASHBOARD
   -------------------
   # Modalità sviluppo (per test):
   npm run dev
   
   # Modalità produzione con PM2:
   pm2 start npm --name "proxy-dashboard" -- run dev
   pm2 save

5. ACCEDI ALLA DASHBOARD
   ----------------------
   Apri browser: http://IP_VPS:5000

===============================================================================
CONFIGURAZIONE CONNESSIONI ATTIVE (OPZIONALE)
===============================================================================

Per vedere le connessioni attive in tempo reale, configura nginx_status:

sudo nano /etc/nginx/conf.d/status.conf

Aggiungi:

server {
    listen 127.0.0.1:80;
    server_name localhost;

    location /nginx_status {
        stub_status on;
        access_log off;
        allow 127.0.0.1;
        deny all;
    }
}

Poi:
sudo nginx -t
sudo systemctl reload nginx

NOTA: Se non configuri nginx_status, la dashboard userà automaticamente 
      'ss' per contare le connessioni sulla porta 8880.

===============================================================================
PERMESSI (SE NON SEI ROOT)
===============================================================================

Se esegui come utente non-root, configura sudoers:

sudo visudo

Aggiungi (sostituisci 'dashboard' con il tuo utente):

dashboard ALL=(ALL) NOPASSWD: /bin/systemctl status nginx
dashboard ALL=(ALL) NOPASSWD: /bin/systemctl status fail2ban
dashboard ALL=(ALL) NOPASSWD: /bin/systemctl status mariadb
dashboard ALL=(ALL) NOPASSWD: /bin/systemctl start nginx
dashboard ALL=(ALL) NOPASSWD: /bin/systemctl stop nginx
dashboard ALL=(ALL) NOPASSWD: /bin/systemctl restart nginx
dashboard ALL=(ALL) NOPASSWD: /bin/systemctl reload nginx
dashboard ALL=(ALL) NOPASSWD: /bin/systemctl start fail2ban
dashboard ALL=(ALL) NOPASSWD: /bin/systemctl stop fail2ban
dashboard ALL=(ALL) NOPASSWD: /bin/systemctl restart fail2ban
dashboard ALL=(ALL) NOPASSWD: /usr/bin/fail2ban-client *

Permetti lettura file:
sudo chmod 644 /etc/nginx/*.conf
sudo chmod 644 /etc/fail2ban/jail.local
sudo chmod 644 /etc/fail2ban/filter.d/*.conf

VERIFICA FUNZIONAMENTO

===============================================================================
TROUBLESHOOTING
===============================================================================

PROBLEMA: Non vedo gli IP bannati
SOLUZIONE: Verifica che fail2ban sia installato e attivo
          sudo systemctl status fail2ban

PROBLEMA: Connessioni attive sempre a 0
SOLUZIONE: Verifica che nginx ascolti sulla porta 8880
          ss -tnlp | grep 8880
          
          La dashboard usa automaticamente fallback:
          ss -tn state established "( sport = :8880 )"

PROBLEMA: Tab Firewall non mostrano contenuto
SOLUZIONE: I file vengono creati automaticamente al primo accesso.
          Se persistono problemi, verifica permessi:
          sudo chmod 644 /etc/nginx/*.conf

PROBLEMA: Errore permessi
SOLUZIONE: Esegui come root oppure configura sudoers (vedi sopra)
